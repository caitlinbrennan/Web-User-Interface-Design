Web Architecture

Web architecture is the structure that the website is built on. It is important to ensure that everything runs smoothly, quickly and efficiently

Web Servers

Web servers serve files that make up web pages to users browsers in response to requests.

Web Protocols

HTTP:
(HyperText Transfer Protocol)
- This is the foundation of data communication, it defines the rules for requesting and transferring web resources, for example, images and stylesheets.
HTTPS:
(HyperText Transfer Protocol Secure)
- It uses encryption (SSL/TLS) to protect data exchanged with clients and servers.
TCP/IP:
(Transmission Control Protocol / Internet Protocol)
- TCP looks after communication between application software and your network software - it breaks down data in packets before sending, then they will be re-assembled when they arrive at the destination.
- IP looks after "routing" each packet to the destination given - this is the communication between two computers.
FTP:
(File Transfer Protocol)
- It enables the transfer of files from the user's local machine to a web browser

Web Concepts

URLS and DNS

A URL is a web address that identifies resources - they consist of a protocol, a domain name, and a path (there can also be sub-domains, folders, and resources included)

The Domain Name System (DNS) translates human-readable domain names into IP addresses - allowing the browser to locate web servers by name

Top-Level Domains are the last segment of a domain name - some generic types include:
      .ie
      .gov
      .com

Cookies and Browsers

Cookies are small data pieces sent by web servers to client's browsers - they're used for identification / tracking user info

Web Browser Chrome Safari Firefox
Engine Chromium Webkit Gecko

The differences these engines can have include having different javascript engines / varied DOM support / all developed independent of each other

Caching is storing copies of frequently accessed web resources to reduce the loas on servers - faster load time

Internet Vs Web

- The internet - short for Interconnected Network - refers to the physical network that links computers around the world - infrastructure of network servers / communication linked between them

- The World Wide Web is the info system available via the internet - tranfers documents using HTTP protocol

Client-Server Communication

- The request-response model refers to communication between the client and the web server - the client sends HTTP request, the server gives info

- A stateless protocol is when each request is independent of each other

- This is two-way communication - it is used for live updates / gaming

Client-Side Tech

      HTML: creates structure
      CSS: styles
      Javascript: increases activity
